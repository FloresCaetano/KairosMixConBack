1. Capa de Dominio (Domain)Es el corazón del software. No debe tener dependencias de frameworks (como Spring) ni de bases de datos.  Entidades: Clases puras de Java (POJOs) que representan tus objetos de negocio.  Lógica de Validación (Correctness): Implementa validaciones dentro de la entidad para asegurar que el estado del objeto siempre sea válido. Por ejemplo, lanzar una excepción si se intenta completar una tarea ya cancelada.  Puertos de Salida (Output Ports): Define interfaces (como TaskRepositoryPort) que describan qué acciones necesita el dominio, pero sin decir cómo se hacen.  2. Capa de Aplicación (Application)Aquí reside la lógica de los procesos del sistema.  Casos de Uso (UseCases): Clases que orquestan el flujo de datos. Deben usar @Transactional para asegurar la Integridad de las operaciones en la base de datos.  Inyección de Dependencias: Utiliza final y @RequiredArgsConstructor de Lombok para inyectar los puertos, lo que facilita la Testabilidad.  Reusabilidad: Esta capa debe ser agnóstica; podrías cambiar la base de datos o el controlador sin tocar un solo caso de uso.  3. Capa de Infraestructura (Infrastructure)Contiene los detalles técnicos y adaptadores.  Persistencia: Implementaciones de los puertos de salida usando Spring Data JPA. Aquí aseguras la Confiabilidad (Reliability) y persistencia de los datos.  Controladores REST (Interfaces):Usa @RestController y mapeos claros como @PostMapping.  Validación de Entrada: Usa @Valid y @RequestBody con DTOs para asegurar que los datos que entran sean correctos (Integrity).  Mapeo: Utiliza una clase Mapper para convertir de DTO a Entidad de Dominio y viceversa, manteniendo las capas separadas.  4. Calidad y Pruebas (Testability & Reliability)El código no está completo sin su validación automatizada.  Pruebas Unitarias: Debes usar JUnit 5 y Mockito.  Estructura de Prueba: Sigue el patrón Arrange-Act-Assert.  Cobertura: La guía exige al menos un 85% de cobertura en las capas de Dominio y Aplicación.  5. Configuración y Automatización (DevSecOps)pom.xml: Debe incluir dependencias para Web, JPA, Validation y Lombok. Usa Java 25 para aprovechar los Virtual Threads y mejorar la Eficiencia.  Pipeline CI/CD: Crea un archivo .yml (ej. GitHub Actions) que automatice:Compilación y Test con Maven (mvn clean verify).  Análisis estático con SonarQube para detectar Bugs y Code Smells.  Escaneo de seguridad (SAST).  Docker: El proyecto debe ser "dockerizable" para garantizar la Portabilidad.  6. Motor de Calidad (Quality Scoring Engine)Este es un requerimiento especial del proyecto. AHORA LA APLICACION DE MCALL Debes crear una clase en Java que reciba las métricas (cobertura, bugs, vulnerabilidades) y calcule un puntaje final basado en pesos específicos:  Correctitud (Correctness): Peso 0.3.  Testabilidad (Testability): Peso 0.3.  Mantenibilidad (Maintainability): Peso 0.2.  Integridad (Integrity): Peso 0.2.
